# 管子检测与追踪阶段总结

## 1. 当前方向

- 目标是从一堆相似管子中检测出每一根管子。
- 用户可以选中其中一根，将它注册为目标。
- 注册后系统尽量在同一摄像头内持续锁定这根管子。
- 如果目标移动到另一个摄像头，后续希望仍能识别为同一根。

当前技术路线：

- 摄像头采集
- YOLO 检测管子
- 点击或按钮注册目标
- 提取目标特征
- 单摄像头内追踪
- 跨摄像头相似度匹配

## 2. 上周分析出的主要挑战

- 管子外观太相似：颜色、形状、长度都接近，单靠外观很难判断是不是同一根。
- 堆叠和遮挡严重：很多管子只露出一部分，容易漏检，也容易把几根管子框成一根。
- 拿取过程中变化大：手拿管子时，角度、光照、露出部分都会变化，目标容易丢失。
- 跨摄像头差异明显：不同摄像头的角度、亮度、画面大小不同，同一根管子看起来可能不一样。
- 检测框不稳定会影响追踪：如果 YOLO 框的位置变化很大，后面的 ID 追踪也会不稳定。
- 注册目标后仍可能跳框：相似管子太多时，G001 可能跳到旁边的管子上。

## 3. 当时提出的处理方法

- 先用 YOLO 做目标检测，不再只依赖运动检测。
- 训练自己的管子/铅笔模型，让模型认识当前目标。
- 标注时要按“每一根管子”单独标，而不是把一堆管子框成一个整体。
- 先解决单摄像头内稳定追踪，再做跨摄像头接力。
- 注册目标后，只重点追踪被选中的那一根。
- 后续可以考虑 ByteTrack、BoT-SORT、DeepSORT 或 ReID 特征增强。
- 如果使用 RealSense，可以进一步利用深度信息辅助判断遮挡和前后位置。

## 4. 目前已经完成的工作

- 完成了基础 CrossCamReID 程序，可以运行摄像头检测和追踪流程。
- 支持 1 个、2 个、3 个摄像头启动。
- 增加了摄像头选择界面，双击 `run_crosscam.bat` 可以选择摄像头。
- 默认优先选择两个外接摄像头，也可以手动指定 `-CameraIndexes "1,3"`。
- 支持画面左右交换和镜像翻转。
- 接入了 YOLO 检测模式。
- 已整理之前拍摄的训练图片，并训练出了 `runs_yolo\pipe_yolov8n\weights\best.pt`。
- 增加了 `PipeMode`，可以加载训练后的管子模型。
- 支持点击检测框注册目标，注册后目标会作为 `G001` 追踪。
- 默认隐藏轨迹线，只保留当前目标框和编号。
- 增加了日志记录和分析脚本，用来判断是否出现 ID 跳变或跨摄像头接力。
- 已提交并推送到 GitHub。

## 5. 当前效果

- 摄像头选择和启动流程已经比较稳定。
- 程序能用训练后的 YOLO 模型检测部分管子目标。
- 能通过点击检测框或按钮注册某一根目标。
- 注册后可以初步显示 `G001` 并进行跟踪。
- 但在复杂堆叠、遮挡、相似管子较多时，锁定还不够稳定。

## 6. 还需要继续做的工作

- 补充更多真实管子场景数据。
- 重点拍堆叠、遮挡、只露头、只露侧面、手拿出来、移动模糊等情况。
- 重新标注并训练 YOLO，提高每一根管子的检测准确率。
- 优化单摄像头内目标锁定，减少 G001 跳到别的管子上。
- 增强 ReID 特征，提高相似管子之间的区分能力。
- 做固定测试：选中一根管子，从堆里拿出来，观察是否一直保持 G001。
- 评估 RF-DETR 作为可选检测后端，和现有 YOLO 做效果对比。
- 后续再继续优化跨摄像头接力效果。

## 7. 简单结论

- 上周的判断和现在的总结方向一致。
- 之前主要是在分析挑战和技术路线。
- 现在已经把摄像头选择、YOLO 接入、训练模型加载、目标注册和基础追踪流程做出来了。
- 下一步重点不是再改摄像头流程，而是提高检测准确率和注册目标后的稳定追踪能力。
