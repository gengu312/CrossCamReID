# 四阶段完善路线

## 总目标

系统要解决的问题不是只判断画面里有没有管子，而是在一堆相似管子中尽量检测出每一根，并在拿取、遮挡、换摄像头时保持同一根管子的身份。

当前推荐路线是：

```text
YOLO 管状物检测
-> 单摄像头多目标追踪
-> 跨摄像头 ReID 特征匹配
-> RealSense 深度和空间位置辅助
```

## 阶段 1：先把每根管子检测稳定

目标：让系统能在堆叠铅笔/管子中尽量框出每一个可见目标。

当前已有：

- `-PipeMode` 会自动使用训练好的 `runs_yolo\pipe_yolov8n\weights\best.pt`。
- 支持 YOLO 检测静止目标，不再只依赖运动检测。
- 支持点击检测框注册要追踪的目标。
- 本阶段已让 YOLO 检测复用管状物几何过滤参数，过滤过大、过短、过粗的误检框。

继续要做：

- 补充堆叠、遮挡、手拿取、只露头、只露侧面的照片。
- 标注时按“每一根可见管子/铅笔”单独框，不要只框整堆。
- 每次训练后用验证集图片检查漏检、框偏大、把多根框成一根的问题。

建议命令：

```powershell
.\run_crosscam.bat -CamA 1 -CamB 2 -Backend dshow -PipeMode
.\evaluate_pipe_yolo.bat
```

## 阶段 2：增强单摄像头内多目标追踪

目标：减少同一个摄像头内 ID 跳变，避免同一根管子移动一下就变成新的 `Gxxx`。

处理方法：

- 在当前轻量 tracker 上加入运动预测，短暂遮挡时继续沿着原方向寻找。
- 后续可接入 ByteTrack、BoT-SORT 或 DeepSORT。
- 记录 ID 切换次数，不能只靠肉眼判断效果。

## 阶段 3：增强跨摄像头 ReID

目标：同一根管子从一个摄像头移到另一个摄像头时，尽量保持同一个全局 ID。

处理方法：

- 当前先用颜色、边缘、形状等轻量特征做相似度。
- 后续可以接入 ResNet、CLIP、DINOv2 这类图像特征。
- 对注册目标保存多张模板，覆盖正面、侧面、手拿取、光照变化。

## 阶段 4：利用 RealSense 深度辅助

目标：在堆叠、遮挡、管子外观相似时，增加空间信息，降低纯 RGB 判断的不稳定。

处理方法：

- 使用深度估计管子的前后层次和距离变化。
- 对拿取动作记录目标从堆中离开的空间轨迹。
- 用 RGB 检测结果加深度位置共同判断目标是否连续。

## 当前验收重点

短期演示先看三件事：

- 堆叠铅笔/管子能不能被检测出多个框。
- 点击某一根后，日志是否出现 `已注册目标为 G001`。
- 拿到另一个摄像头后，是否还能出现同一个 `G001`，如果变成新编号，则说明检测到了物体但 ReID 接力没接上。
