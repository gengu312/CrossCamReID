## 视频测试

六段视频统一使用 `imgsz=640`、`IoU=0.45`，每 10 帧抽样一次。

| 场景 | 原模型平均框数 | 增量模型平均框数 | 说明 |
| --- | ---: | ---: | --- |
| 静止堆叠 | 2.28 | 3.88 | 增量模型能找到更多静止目标 |
| 拿出一根 | 3.91 | 4.19 | 检测数量略有提高 |
| 手部遮挡 | 4.48 | 4.76 | 框数提高，连续性仍需继续优化 |
| 移动旋转 | 4.75 | 4.05 | 快速变化场景没有全面提升 |
| 单路交接视频 | 有效帧率 0.824 | 有效帧率 0.843 | 增量模型略好 |
| 负样本 | 0 误检 | 0 误检 | `conf=0.25` 下均通过 |

`conf=0.20` 能把静止堆叠平均框数提高到 6.50，但负样本误检帧率为 0.025，因此默认继续使用 `0.25`。

增量模型检测更全，调整 `TargetSwitchMargin=0.15` 后锁定稳定性接近原模型，并减少了目标框最大位移。因此 PipeMode 优先使用增量模型，同时保留原权重作为回退。

## 当前限制和后续方向

- 本轮图片主要来自同一桌面和同一种绿色铅笔，不能替代真实铁管现场数据。
- 自动标签属于保守近似标签，不等同于完全人工校正真值。
- 标准水平框在斜放、交叉和紧密平行时会高度重叠，后续可评估 YOLO OBB 或实例分割。
- `05_handoff` 目前只有 `camera_a.mp4`，还不能验证真实双路同步跨摄像头接力。
- 下一批优先采集不同背景、光照、距离、锈蚀纹理和真实铁管端面，并保留独立现场测试集。
