# 铅笔模拟铁管的数据采集计划

目标是先把铅笔当成 `pipe` 类别，训练一个能识别静止和移动管状物的 YOLO 模型。后面换成真实铁管时，项目结构不变，只替换数据和模型。

## 第一批照片数量

先拍 100 到 200 张，不要一开始拍太多。第一批的目标是验证训练流程和检测效果。

建议比例：

- 单根铅笔：20 张
- 2 到 5 根铅笔并排或堆叠：40 张
- 6 根以上密集堆叠：30 张
- 手拿移动或刚放下：30 张
- 两个摄像头分别拍摄：每个摄像头至少 40 张
- 失败/干扰场景：20 张，比如手、桌面杂物、反光、阴影

## 必须覆盖的变化

- 不同角度：横放、斜放、竖放、交叉摆放。
- 不同距离：近景、中景、远一点的全局画面。
- 不同光照：正常光、偏暗、侧光、有阴影。
- 不同背景：桌面、纸面、金属/塑料背景。
- 不同遮挡：一部分被手挡住、一根压住另一根。
- 不同状态：静止、移动、进入画面、离开画面。

## 拍摄方式

1. 固定摄像头，不要一直手持摄像头移动。
2. 每次改变铅笔摆放后拍几张。
3. 同一堆铅笔不要只拍一张，轻微改变角度和距离再拍。
4. 图片里要有“没有铅笔”的负样本，帮助模型减少误检。
5. 文件按场景整理，例如：

```text
dataset_raw/
  cam1_single/
  cam1_stack/
  cam1_hand_move/
  cam2_single/
  cam2_stack/
  negative/
```

## 标注建议

第一版先做 YOLO detect，标矩形框即可。

具体训练目录和命令见：

```text
docs/yolo_training_guide.md
```

类别名统一用：

```text
pipe
```

即使用铅笔模拟，也不要标成 `pencil`，这样后面迁移到真实铁管时概念一致。

标注规则：

- 每一根可见铅笔都标一个框。
- 被遮挡严重、只露出很小一段的目标可以先不标。
- 框尽量贴住可见管身，不要框太多背景。
- 多根重叠时，能区分几根就标几根。

第二版再考虑 YOLO segmentation 或 OBB：

- 如果只需要检测和跟踪，继续用 detect。
- 如果要贴合每根管子的轮廓，升级 segmentation。
- 如果管子大量斜放，升级 OBB 旋转框。

## 什么时候扩充数据

第一批 100 到 200 张训练后，如果出现下面问题，再扩到 300 到 800 张：

- 静止铅笔漏检。
- 多根堆叠时只能识别成一大块。
- 换摄像头或换光照后效果明显下降。
- 手或桌面纹理被误识别成 pipe。

扩充时优先补模型失败的场景，不要只拍好看的重复照片。
