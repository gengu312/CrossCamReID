# 项目现状与改进路线

## 1. 当前结论

项目当前技术路线是可行的：

```text
管子检测
-> 单摄像头目标锁定
-> 跨摄像头目标重识别
-> 深度和空间约束辅助
```

现阶段应优先解决前两项。检测框如果漏检、合并或频繁跳动，后续跟踪和跨摄像头匹配只能缓解问题，无法从根本上保证目标身份稳定。

## 2. 已有能力

- 支持 1 到 3 个摄像头，并可在启动界面选择设备。
- 支持画面左右交换、水平翻转和检测区域设置。
- 支持 `motion`、YOLO 和 RF-DETR 三种检测入口。
- 已训练第一版 `pipe` YOLO 模型，可检测部分静止和移动目标。
- 支持点击检测框注册目标，并用 `G001` 表示当前锁定目标。
- 支持单摄像头跟踪、目标短时丢失保护和跨摄像头 ID 接力。
- 支持保存目标样本和事件日志，并用脚本检查跳框、目标切换和接力结果。

## 3. 当前数据和效果

当前正式 YOLO 数据集包含：

- 训练集：121 张图片、595 个目标框。
- 验证集：30 张图片、159 个目标框。
- 总计：151 张图片、754 个目标框。

现有候选模型在当前验证集上的较好结果约为：

- Precision：0.68。
- Recall：0.70。
- 平均 IoU：0.67。

该结果可以用于流程演示，但复杂堆叠场景仍有漏检、误检和框不稳定问题。当前验证集规模较小，且主要来自同一批铅笔素材，不能代表真实铁管场景的最终效果。

## 4. 主要问题

### 4.1 检测数据不足

当前数据还没有充分覆盖复杂堆叠、只露头、只露侧面、严重遮挡、手拿取、运动模糊、光照变化和背景干扰。铅笔数据可以验证流程，但后续必须使用真实铁管重新采集和训练。

### 4.2 检测和身份识别是两个问题

当前 `pipe` 标签只教模型判断画面中是否存在管子，不能教会模型判断“这是其中哪一根”。单根管子的身份需要视频时序、目标跟踪、外观特征和空间位置共同判断。

### 4.3 当前跟踪方法仍较轻量

现有 tracker 主要结合位置、IoU、颜色、边缘和纹理进行匹配。它适合验证系统流程，但面对大量外观相近的管子、遮挡和快速移动时，仍可能出现 ID 跳变。

### 4.4 相同管子存在识别上限

如果多根铁管外观几乎完全相同，两个摄像头又没有重叠视野，纯 RGB 图像无法保证 100% 确认身份。要求工业级强保证时，需要增加深度、重叠视野、运动时间约束，或者使用色标、二维码、RFID 等辅助标识。

## 5. 改进优先级

### 第一优先级：建立可重复测试

目标是让同一组素材可以重复运行，公平比较不同模型和参数。

需要完成：

- 支持 1 到 3 路本地视频作为输入。
- 默认同步逐帧读取，任一路视频结束时停止。
- 复用现有检测、注册、跟踪和日志分析流程。
- 固定 5 到 10 组测试场景，保存每次运行的质量指标。

建议测试场景：

- 堆叠目标保持静止。
- 从相邻目标中拿出一根。
- 手部短暂遮挡目标。
- 目标快速移动或产生运动模糊。
- 目标离开一个画面后进入另一个画面。
- 没有管子的负样本场景。

### 第二优先级：补充数据并重新训练

- 将有效图片扩充到 300 到 800 张。
- 优先补充模型失败的场景，不重复拍摄大量相似照片。
- 每一根清晰可见的管子都单独标注。
- 自动标注只生成候选框，训练前必须人工复查。
- 按拍摄批次划分训练集、验证集和测试集，避免相邻连拍图片分到不同集合。
- 保留一套不参与训练的固定测试集。

### 第三优先级：升级单摄像头跟踪

- 使用现有检测结果对比 ByteTrack 和 BoT-SORT。
- 记录 `G001` 跳框、临时丢失和 ID 切换次数。
- 如果框级跟踪仍不稳定，再评估 SAM 2 等视频目标分割方案。

### 第四优先级：处理细长和重叠目标

- 斜放管子包含过多背景时，评估 YOLO OBB 旋转框。
- 必须分离严重重叠轮廓时，评估实例分割。
- 使用同一数据集和同一测试集比较 YOLO 与 RF-DETR，不根据单次画面判断模型优劣。

### 第五优先级：增强跨摄像头和深度判断

- 为注册目标保存不同角度和不同光照下的可靠模板。
- 使用深度特征或专用 ReID 特征替换单纯颜色纹理匹配。
- 加入目标离开和进入不同摄像头的时间、方向和区域约束。
- 使用 RealSense 深度判断目标是否从堆中被拿出，以及距离变化是否连续。

## 6. 阶段验收标准

### 检测验收

- 在独立测试集上分别统计 Precision、Recall 和 IoU。
- 第一阶段目标是 Precision 和 Recall 接近或超过 0.85。
- 重点检查多根合并成一个框、只露部分时漏检和手部误检。

### 单摄像头锁定验收

- 点击一根目标后连续移动 20 秒。
- `G001` 不切换到相邻管子。
- 注册目标大跳次数为 0。
- 同一动作重复 5 次，至少 4 次完整锁定成功。

### 跨摄像头验收

- 固定完成 10 次跨画面移动。
- 统计目标是否继续使用同一个 `G001`。
- 同时记录失败发生在检测、单摄像头跟踪还是跨摄像头匹配阶段。

## 7. 当前实施顺序

```text
增加离线视频回放
-> 录制固定测试素材并建立当前基线
-> 补充失败场景数据并重新训练 YOLO
-> 对比成熟跟踪器
-> 再评估 RF-DETR、OBB、实例分割和深度 ReID
```

第一步完成后，项目可以在不占用真实摄像头的情况下重复复现同一问题，也能用日志判断每次优化是否真正减少漏检和跳框。
